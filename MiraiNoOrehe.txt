


----------未来のオレへ向けた当アプリ簡易説明--------------

Python実行出来ないとダメなので実行環境を整えましょう
DLとかの説明面倒なのでここ↓を見てくれ
https://www.sejuku.net/blog/33294

でPythonを実行出来るようになったら、

run.pyをダブルクリックなりなんなりで実行すると
上手くいけばlocalhost:5000にアクセスしろ出るからそこにアクセス
すると使える　わーお

機能は配信が見たいVtuber追加、Vtuber一覧、配信予定一覧、更新ボタン、削除のみ
Vtuber一覧と配信予定一覧の隣にあるチェックボックスに、
チェック入れて上の削除押せばチェックした項目が消える
ちなみに各ボタン押した後はURLちょっと変わるので、
F5とか標準の更新する時はちゃんとlocalhost:5000で更新しようね(色々面倒なため)

コード変えたりする時は
コマンドプロンプトとかでこいつらをまずインストールしよう！
pip3 install Flask
pip3 install beautifulsoup4
pip3 install sqlalchemy


-----------------ここからフォルダとか説明---------------------

【run.pyくん】

鯖構築ファイル
こいつで鯖構築後appフォルダ直下にあるapp.pyを呼び覚ましてる
なのでapp.pyくんが調子悪いと鯖立ててくれない
ちなみにコードの記述内容的に初手の読み込む段階でネット環境必須なので、
オフラインだとapp.py調子悪いやん！って鯖立ててくれない


【appフォルダ】

中にあるapp.pyはページ生成だとか動きだとかを司る神
ここにメイン処理色々書いてる(読み込んだ時とかボタン押した時とかそこらへん)

ちなみにこのフォルダ直下にある、
template フォルダにはhtmlが単体で幽閉されてて
static フォルダにはcssと画像フォルダがある
__pycache__ フォルダは…何それ知らん…怖…
っていうのは冗談でキャッシュ保存とかしてくれていて、
これによりコンパイルとか経ずに実行早くしてくれてるらしい
詳しくはここ↓
https://qiita.com/Kobayashi2019/items/03e31ee50b924f428e71


【modelsフォルダ】
主にDB取り扱ってるフォルダ
__init__.pyくんは添えるだけ
詳しくはここ↓
https://python.ms/class/#%E3%81%AF%E3%81%98%E3%82%81%E3%81%AB

database.pyはDB作成。models.pyはテーブル作成とかしてる
vtuber.dbはdatabase.pyによって作成されたDBファイルで、
sqlite3.exe三兄弟はこのDBファイルをcmdで直接編集するためのもの
ここではsqlite3 vtuber.pyでいじれる
コマンドに関してはここ↓
https://www.dbonline.jp/sqlite/
